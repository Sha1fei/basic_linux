# Команда `chmod`

`chmod` (change mode) меняет **права доступа** к файлам и каталогам: кто может читать, записывать и выполнять.

Утилита обычно находится в `/bin/chmod` или `/usr/bin/chmod`.

## Права доступа

Права задаются для трёх категорий:


| Категория          | Буква | Кто                        |
| ------------------ | ----- | -------------------------- |
| user (владелец)    | `u`   | владелец файла             |
| group (группа)     | `g`   | члены группы файла         |
| others (остальные) | `o`   | все остальные пользователи |


Три вида прав:


| Право                | Буква | Для файла               | Для каталога              |
| -------------------- | ----- | ----------------------- | ------------------------- |
| read (чтение)        | `r`   | открыть и прочитать     | просмотреть список (`ls`) |
| write (запись)       | `w`   | изменить содержимое     | создавать и удалять файлы |
| execute (выполнение) | `x`   | запустить как программу | войти в каталог (`cd`)    |


Пример из `ls -l`:

```text
-rwxr-xr--  1 user group  1024 Jun 15 10:00 script.sh
```

Здесь `rwxr-xr--` означает: владелец — чтение, запись, выполнение; группа — чтение и выполнение; остальные — только чтение.

## Базовое использование

Синтаксис:

```text
chmod [опции] режим файл...
```

Символьная запись (буквы):

```bash
chmod u+x script.sh      # добавить выполнение владельцу
chmod g-w file.txt       # убрать запись у группы
chmod o=r file.txt       # остальным только чтение
chmod a+r file.txt       # всем добавить чтение (a = all)
chmod u+x,g+x script.sh  # несколько изменений через запятую
```

Октальная запись (цифры):

```bash
chmod 755 script.sh
chmod 644 file.txt
chmod 600 secret.txt
```

## Октальная нотация

Каждая цифра — сумма прав для одной категории (владелец, группа, остальные):


| Цифра | Права | Расчёт              |
| ----- | ----- | ------------------- |
| 0     | `---` | нет прав            |
| 1     | `--x` | только выполнение   |
| 2     | `-w-` | только запись       |
| 3     | `-wx` | запись + выполнение |
| 4     | `r--` | только чтение       |
| 5     | `r-x` | чтение + выполнение |
| 6     | `rw-` | чтение + запись     |
| 7     | `rwx` | все права           |


Частые значения:


| Режим | Права       | Типичное применение              |
| ----- | ----------- | -------------------------------- |
| `644` | `rw-r--r--` | обычный файл                     |
| `755` | `rwxr-xr-x` | исполняемый файл или каталог     |
| `600` | `rw-------` | приватный файл (только владелец) |
| `700` | `rwx------` | приватный каталог или скрипт     |


## Символьные операторы


| Оператор | Значение                                        |
| -------- | ----------------------------------------------- |
| `+`      | добавить права                                  |
| `-`      | убрать права                                    |
| `=`      | установить права точно (остальные сбрасываются) |


Примеры:

```bash
chmod +x script.sh       # добавить выполнение всем (эквивалент a+x)
chmod u=rwx,go=rx dir/   # каталог: владелец всё, остальным чтение и вход
chmod -R u+rwX dir/      # рекурсивно; X — выполнение только для каталогов
```

`X` (заглавная) добавляет выполнение только там, где оно уже есть у кого-то из категорий, или для каталогов — удобно при рекурсивной смене прав.

## Полезные опции


| Опция              | Описание                                                            |
| ------------------ | ------------------------------------------------------------------- |
| `-R`               | Рекурсивно: применить ко всем файлам внутри каталога                |
| `-v`               | Показать, какие файлы изменены                                      |
| `-c`               | Показать только изменённые файлы (как `-v`, но без «без изменений») |
| `--reference=файл` | Взять права с другого файла                                         |


```bash
chmod -R 755 project/
chmod --reference=template.sh new_script.sh
```

## Примеры

Сделать скрипт исполняемым:

```bash
touch script.sh
chmod +x script.sh
ls -l script.sh
# -rwxr-xr-x ... script.sh
```

Ограничить доступ к файлу с паролем:

```bash
chmod 600 .env
ls -l .env
# -rw------- ... .env
```

Права на каталог проекта:

```bash
mkdir -p project/src
chmod 755 project
ls -ld project
# drwxr-xr-x ... project
```

Рекурсивно для дерева файлов:

```bash
chmod -R u+rwX,go+rX project/
```

Скопировать права с существующего файла:

```bash
chmod --reference=old.sh new.sh
```

## Важно

- Менять права может **владелец** файла или **root**.
- Права на каталог важны отдельно: без `x` на каталог нельзя войти в него, даже если у файла внутри есть `r`.
- `chmod` меняет только права доступа; владельца и группу меняют `chown` и `chgrp`.
