# Файл `/etc/passwd`

`/etc/passwd` — системный файл со **списком пользователей** Linux.

Обычный пользователь может **читать** этот файл. Пароли здесь **не хранятся** (они в `/etc/shadow`).

## Просмотр

Показать весь файл:

```bash
cat /etc/passwd
```

Первые строки:

```bash
head /etc/passwd
```

Найти конкретного пользователя:

```bash
grep alex /etc/passwd
```

## Формат строки

Пример:

```text
root:x:0:0:root:/root:/bin/bash
```

| Поле | Пример | Значение |
| ---- | ------ | -------- |
| 1 | `root` | имя пользователя (логин) |
| 2 | `x` | пароль: `x` значит «смотри в `/etc/shadow`» |
| 3 | `0` | uid — ID пользователя |
| 4 | `0` | gid — ID основной группы |
| 5 | `root` | комментарий (часто полное имя) |
| 6 | `/root` | домашний каталог |
| 7 | `/bin/bash` | shell по умолчанию |

Разделитель полей — двоеточие `:`.

## Примеры

Посмотреть пользователя `root`:

```bash
grep ^root: /etc/passwd
# root:x:0:0:root:/root:/bin/bash
```

## Служебные пользователи

Не все строки — «живые» люди. Многие записи нужны для сервисов:

```text
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin
```

У таких пользователей shell часто `/usr/sbin/nologin` — войти под ними нельзя.

## `/etc/passwd` и `/etc/shadow`

| Файл | Содержимое | Кто может читать |
| ---- | ---------- | ---------------- |
| `/etc/passwd` | имена, uid, домашние каталоги | все |
| `/etc/shadow` | хэши паролей | обычно только root |

```bash
cat /etc/shadow
# Permission denied   (если вы не root)
```

## Связанные команды и файлы

- [/etc/shadow](shadow.md) — хэши паролей
- `id` — uid и группы текущего пользователя
