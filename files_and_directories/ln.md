# Жёсткие и символические ссылки (`ln`)

В Linux один и тот же файл может иметь **несколько имён**. Для этого используют ссылки:

- **жёсткая ссылка** (hard link) — ещё одно имя того же файла на диске;
- **символическая ссылка** (soft link, symlink) — отдельный файл-указатель на другой путь.

Ссылки создают командой `ln`. Утилита обычно находится в `/bin/ln` или `/usr/bin/ln`.

## Жёсткая ссылка (hard link)

Жёсткая ссылка — это **дополнительное имя** с тем же **inode**, что и у исходного файла. Данные на диске одни, имён может быть несколько.

```bash
echo "hello" > file
ln file hardlink
ls -li file hardlink
```

Пример вывода:

```text
1234567 -rw-r--r-- 2 user user 6 Jun 15 10:00 file
1234567 -rw-r--r-- 2 user user 6 Jun 15 10:00 hardlink
```

Обратите внимание:

- одинаковый **inode** (`1234567`);
- в колонке **Links** стоит `2` — две жёсткие ссылки на один файл.

Изменение через любое имя видно из другого:

```bash
echo "world" >> hardlink
cat file
# hello
# world
```

Файл исчезает с диска только когда удалены **все** жёсткие ссылки:

```bash
rm file
cat hardlink          # данные ещё на месте
rm hardlink           # теперь inode освобождён
```

### Ограничения жёстких ссылок

| Ограничение | Пояснение |
| ----------- | --------- |
| Только внутри одного раздела | inode уникален в пределах файловой системы |
| Обычно только на файлы | на каталоги жёсткие ссылки не создают (кроме `.` и `..`) |
| Нельзя на несуществующий файл | цель должна уже существовать |

## Символическая ссылка (soft link)

Символическая ссылка — это **отдельный файл** со своим inode, в котором записан **путь** к другому файлу или каталогу.

```bash
ln -s file softlink
ls -li file softlink
```

Пример вывода:

```text
1234567 -rw-r--r-- 1 user user 6 Jun 15 10:00 file
1234568 lrwxrwxrwx 1 user user 4 Jun 15 10:01 softlink -> file
```

Признаки symlink в `ls -l`:

- тип `l` в начале прав (`lrwxrwxrwx`);
- стрелка `->` и путь к цели;
- **свой inode**, отличный от цели.

Проверить, куда указывает ссылка:

```bash
readlink softlink
# file

ls -l softlink
```

### Особенности символических ссылок

| Свойство | Пояснение |
| -------- | --------- |
| Можно на каталоги | удобно для «ярлыков» на папки |
| Можно между разделами | путь может вести на другую ФС |
| Может быть «битой» | если цель удалена, ссылка остаётся, но не работает |
| Размер — длина пути | не размер файла-цели |

```bash
ln -s missing.txt broken
cat broken
# cat: broken: No such file or directory

ls -l broken
# lrwxrwxrwx ... broken -> missing.txt
```

## Сравнение

| | Жёсткая ссылка | Символическая ссылка |
| --- | --- | --- |
| Команда | `ln источник ссылка` | `ln -s цель ссылка` |
| Что хранится | тот же inode | путь к файлу |
| Inode | общий с целью | свой |
| Каталоги | обычно нет | да |
| Другой раздел | нет | да |
| Удаление цели | данные живут, пока есть другие hardlink | ссылка «битая» |
| В `ls -l` | как обычный файл | `l... -> путь` |

## Команда `ln`

Синтаксис:

```text
ln [опции] источник ссылка
ln [опции] источник ... каталог/
ln -s [опции] цель ссылка
```

| Опция | Описание |
| ----- | -------- |
| `-s` | Создать символическую ссылку |
| `-f` | Перезаписать существующую ссылку или файл |
| `-i` | Спрашивать перед перезаписью |
| `-v` | Показать, что создано |
| `-T` | С `-s`: трактовать `ссылка` как файл, а не каталог |

### Примеры

Жёсткая ссылка:

```bash
touch original
ln original hardlink
stat original hardlink
```

Символическая ссылка на файл:

```bash
ln -s /etc/hosts my_hosts
ls -l my_hosts
```

Символическая ссылка на каталог:

```bash
ln -s /var/log logs
cd logs
```

Несколько жёстких ссылок в каталог:

```bash
mkdir links
ln file links/a
ln file links/b
ls -li file links/
```

Абсолютный и относительный путь в symlink:

```bash
ln -s ../data/config.ini app.ini    # относительный — переносимее
ln -s /etc/nginx/nginx.conf nginx.conf   # абсолютный — надёжнее при смене cwd
```

## Команда `readlink`

`readlink` показывает, **куда указывает символическая ссылка** — выводит путь, записанный внутри symlink.

Утилита обычно находится в `/usr/bin/readlink`.

В `ls -l` цель видна после стрелки `->`, но `readlink` удобнее для скриптов и когда нужен только путь без лишнего вывода.

### Базовое использование

```bash
ln -s /etc/hosts my_hosts
readlink my_hosts
# /etc/hosts
```

Синтаксис:

```text
readlink [опции] файл...
```

### Полезные опции

| Опция | Описание |
| ----- | -------- |
| *(без опций)* | Показать путь, на который указывает ссылка |
| `-f` | Развернуть цепочку ссылок до конечного файла (канонический абсолютный путь) |
| `-e` | Как `-f`, но только если все компоненты пути существуют |
| `-n` | Не добавлять перевод строки в конце |
| `-q` | Тихий режим: без сообщений об ошибках |
| `-v` | Подробный вывод |

### Примеры

Одна ссылка — один шаг:

```bash
ln -s file softlink
readlink softlink
# file
```

Цепочка ссылок:

```bash
ln -s file link1
ln -s link1 link2

readlink link2
# link1

readlink -f link2
# /полный/путь/к/file
```

Проверка «битой» ссылки:

```bash
ln -s missing.txt broken
readlink broken
# missing.txt

readlink -e broken
# ошибка: компонент пути не существует
```

### `readlink` и `stat`

| Команда | Результат |
| ------- | --------- |
| `readlink link` | только путь-цель: `file` |
| `stat link` | метаданные **самой ссылки** (inode, права, размер = длина пути) |
| `stat -L link` | метаданные **файла-цели** |

```bash
ln -s /etc/hosts my_hosts

readlink my_hosts
stat my_hosts
stat -L my_hosts
```

`readlink` читает symlink; `stat` описывает файл или ссылку как объект на диске.
