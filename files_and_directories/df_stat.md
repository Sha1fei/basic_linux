# Команды `df` и `stat`

`df` (disk free) показывает **свободное место на файловых системах**.
`stat` выводит **подробные метаданные** конкретного файла или каталога.

Обе утилиты обычно находятся в `/usr/bin/df` и `/usr/bin/stat`.

---

## Команда `df`

`df` отвечает на вопрос: сколько места занято и сколько осталось на смонтированных разделах.

### Базовое использование

Показать все смонтированные файловые системы:

```bash
df
```

Типичный вывод:

```text
Filesystem     1K-blocks    Used Available Use% Mounted on
/dev/sda1       102554624 12345678  90123456  13% /
tmpfs              8123456    12345   8111111   1% /run
```

| Колонка | Значение |
| ------- | -------- |
| `Filesystem` | Устройство или источник монтирования |
| `1K-blocks` | Общий размер в блоках по 1 КБ |
| `Used` | Занято |
| `Available` | Доступно обычным пользователям |
| `Use%` | Процент использования |
| `Mounted on` | Точка монтирования |

### Полезные опции

| Опция | Описание |
| ----- | -------- |
| `-h` | Размеры в удобном виде: `K`, `M`, `G` |
| `-H` | То же, но степени 1000 (как у производителей дисков) |
| `-T` | Показать тип файловой системы (`ext4`, `tmpfs` и т.д.) |
| `-t TYPE` | Только указанный тип, например `-t ext4` |
| `-x TYPE` | Исключить указанный тип |
| `-i` | Показать использование **inode**, а не места на диске |
| `-a` | Показать все файловые системы, включая «фиктивные» с 0 блоками |
| `-l` | Только локальные файловые системы (без NFS и т.п.) |
| `--total` | Добавить строку с итогом (удобно с `-h`) |
| `--output=FIELD,...` | Выбрать колонки вывода |

Опции можно комбинировать:

```bash
df -hT
df -h --total
```

### Примеры

Свободное место в человекочитаемом виде:

```bash
df -h
```

Только корневой раздел:

```bash
df -h /
```

Проверить домашний каталог (если он на отдельном разделе):

```bash
df -h ~
df -h /home
```

Проверить inode (полезно, когда «место есть», а файлы не создаются):

```bash
df -i
df -ih
```

Только ext4-разделы:

```bash
df -hT -t ext4
```

Итог по всем локальным разделам:

```bash
df -h --total -l
```

### Практические заметки

- `df` смотрит на **файловую систему целиком**, а не на размер одного каталога.
- Чтобы узнать, сколько занимает конкретная папка, используют `du` (отдельная тема).
- Процент `Use%` может быть 100%, но `Available` не равен нулю: часть блока зарезервирована для root.

## Команда `stat`

`stat` показывает свойства одного файла или каталога: размер, права, inode, временные метки, устройство.

### Базовое использование

Информация о файле:

```bash
stat file
```

Типичный вывод:

```text
  File: file
  Size: 0               Blocks: 0          IO Block: 4096   regular empty file
Device: 801h/2049d      Inode: 1234567     Links: 1
Access: (0644/-rw-r--r--)  Uid: ( 1000/   user)   Gid: ( 1000/   user)
Access: 2024-06-15 10:00:00.123456789 +0000
Modify: 2024-06-15 10:00:00.123456789 +0000
Change: 2024-06-15 10:00:00.123456789 +0000
 Birth: 2024-06-15 10:00:00.123456789 +0000
```

| Поле | Значение |
| ---- | -------- |
| `File` | Путь к объекту |
| `Size` | Логический размер в байтах |
| `Blocks` | Сколько блоков занято на диске |
| `IO Block` | Размер блока ввода-вывода |
| `Device` | Устройство, где лежит файл |
| `Inode` | Номер inode — уникальный идентификатор внутри раздела |
| `Links` | Число жёстких ссылок |
| `Access` | Права, uid и gid |
| `Access` (время) | **atime** — последнее чтение |
| `Modify` | **mtime** — последнее изменение содержимого |
| `Change` | **ctime** — последнее изменение метаданных (права, владелец) |
| `Birth` | **btime** — время создания (если поддерживается ФС) |

### Полезные опции

| Опция | Описание |
| ----- | -------- |
| `-c FORMAT` | Свой формат вывода (см. `%`-подстановки в `man stat`) |
| `-f` | Информация о **файловой системе**, а не о файле |
| `-L` | Следовать символической ссылке и показать данные цели |
| `-t` | Краткий (terse) вывод |
| `--printf=FORMAT` | Форматированный вывод, как у `-c`, но без перевода строки в конце |

### Примеры

Подробная информация о каталоге:

```bash
stat /etc
```

Сравнить файл и символическую ссылку:

```bash
stat link
stat -L link
```

Краткий вывод:

```bash
stat -t file
```

Только нужные поля:

```bash
stat -c '%n %s %a %U %G' file1 file2
```

Полезные подстановки для `-c`:

| Подстановка | Значение |
| ----------- | -------- |
| `%n` | имя файла |
| `%s` | размер в байтах |
| `%a` | права в восьмеричном виде |
| `%A` | права в виде `rwxr-xr-x` |
| `%U` | имя владельца |
| `%G` | имя группы |
| `%i` | inode |
| `%F` | тип файла |
| `%x` | atime |
| `%y` | mtime |
| `%z` | ctime |
| `%w` | btime |

Информация о файловой системе, где лежит файл:

```bash
stat -f /home
```

Пример вывода `stat -f`:

```text
  File: "/home"
     ID: 80100000000 Namelen: 255     Type: ext2/ext3/ext4
Block size: 4096       Fundamental block size: 4096
Blocks: Total: 25638656   Free: 22567890   Available: 21345678
Inodes: Total: 6553600    Free: 6543201
```

### Метки времени: atime, mtime, ctime

| Метка | Когда меняется | Пример |
| ----- | -------------- | ------ |
| **atime** | при чтении файла | `cat file` |
| **mtime** | при изменении содержимого | запись в файл, `touch -m` |
| **ctime** | при изменении метаданных | `chmod`, `chown`, `touch` без `-m` |

Посмотреть времена через `ls` и `stat`:

```bash
ls -l file
stat file
```

Изменить mtime:

```bash
touch -m file
stat -c '%y' file
```

### `stat -f` и `df`

Обе команды описывают файловую систему, но по-разному:

| Команда | Уровень | Что показывает |
| ------- | ------- | -------------- |
| `df -h /path` | весь раздел | занято/свободно на смонтированной ФС |
| `stat -f /path` | та же ФС | тип, размер блока, inode, блоки total/free/available |

```bash
df -h /home
stat -f /home
```

`df` удобен для быстрой проверки «сколько места осталось».
`stat` — когда нужны детали одного файла или параметры конкретной ФС.
