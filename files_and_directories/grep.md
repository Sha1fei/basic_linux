# Команда `grep`

`grep` (global regular expression print) ищет **строки по шаблону** в файлах или в потоке ввода.

Утилита обычно находится в `/bin/grep` или `/usr/bin/grep`.

`grep` работает с **содержимым** файлов. Чтобы найти саму программу на диске, используют [whereis](whereis.md) или [type](type.md).

## Базовое использование

Найти строки с текстом в файле:

```bash
grep root /etc/passwd
```

Несколько файлов:

```bash
grep error app.log app.old.log
```

Поиск в выводе другой команды (через pipe):

```bash
ls -l | grep txt
ps aux | grep ssh
```

Синтаксис:

```text
grep [опции] шаблон [файл...]
```

Если файлы не указаны, `grep` читает **stdin**:

```bash
grep hello
# вводите строки; Ctrl+D — конец ввода
```

## Полезные опции

| Опция | Описание |
| ----- | -------- |
| `-i` | Без учёта регистра |
| `-v` | Инвертировать: показать строки **без** совпадения |
| `-n` | Номера строк |
| `-c` | Только количество совпадений |
| `-l` | Только имена файлов, где есть совпадение |
| `-L` | Только имена файлов **без** совпадений |
| `-r` | Рекурсивно по каталогу |
| `-R` | Как `-r`, включая символические ссылки |
| `-w` | Искать целое слово |
| `-x` | Совпадение всей строки целиком |
| `-E` | Расширенные регулярные выражения (ERE) |
| `-F` | Точный текст, без regex (быстрее) |
| `-A N` | Показать N строк **после** совпадения |
| `-B N` | Показать N строк **до** совпадения |
| `-C N` | N строк до и после (`-A` и `-B` вместе) |
| `--color=auto` | Подсветка совпадений (часто включена по умолчанию) |
| `-h` | Без имён файлов в выводе |
| `-H` | Всегда показывать имя файла (даже для одного файла) |

Опции можно комбинировать:

```bash
grep -rin error logs/
grep -E 'error|warn' app.log
```

## Примеры

Найти пользователя в `/etc/passwd`:

```bash
grep root /etc/passwd
```

Без учёта регистра:

```bash
grep -i error app.log
```

С номерами строк:

```bash
grep -n "failed" /var/log/auth.log
```

Сколько раз встречается:

```bash
grep -c error app.log
```

В каких файлах есть совпадение:

```bash
grep -l TODO project/*.md
```

Исключить строки:

```bash
grep -v "^#" config.conf    # пропустить комментарии, начинающиеся с #
```

Целое слово (не `errors`, если ищем `error`):

```bash
grep -w error app.log
```

Рекурсивный поиск в каталоге:

```bash
grep -r "password" /etc/nginx/
grep -rn "function" src/
```

Контекст вокруг совпадения:

```bash
grep -C 2 "Exception" app.log
```

## Шаблоны и регулярные выражения

По умолчанию `grep` понимает **базовые** регулярные выражения (BRE). С `-E` — расширенные (ERE).

| Шаблон | Значение |
| ------ | -------- |
| `.` | любой один символ |
| `^` | начало строки |
| `$` | конец строки |
| `*` | ноль или более предыдущих символов |
| `?` | с `-E`: ноль или **один** предыдущий символ; без `-E` — обычный знак `?` |
| `[abc]` | один символ из набора |
| `[0-9]` | цифра |
| `\` | экранирование спецсимвола |

```bash
grep '^root' /etc/passwd          # строки, начинающиеся с root
grep 'bash$' /etc/passwd          # строки, заканчивающиеся на bash
grep -E 'error|warn' app.log      # error или warn
grep -E 'colou?r' words.txt       # color или colour
grep -E '[0-9]{4}-[0-9]{2}' dates.txt
grep -F 'what?' file.txt          # буквальный знак вопроса
```

Если шаблон — обычный текст со спецсимволами, используйте `-F` или экранируйте:

```bash
grep -F 'price=$100' file.txt
grep 'price=\$100' file.txt
```

## Цветной вывод

Подсветка совпадений:

```bash
grep --color=auto error app.log
```

Отключить:

```bash
grep --color=none error app.log
```

## Практические сценарии

Найти IP-адреса в логе:

```bash
grep -Eo '([0-9]{1,3}\.){3}[0-9]{1,3}' access.log
```

Проверить, есть ли строка в файле (код возврата):

```bash
grep -q "OK" status.txt && echo "found"
```

`-q` (quiet) — ничего не печатает, только код выхода: `0` — найдено, `1` — нет.

Поиск в сжатом файле (через `zgrep` или pipe):

```bash
zgrep error app.log.gz
zcat app.log.gz | grep error
```

## `grep`, `whereis` и `find`

| Команда | Что ищет |
| ------- | -------- |
| `grep` | **текст внутри** файлов |
| `whereis` | бинарник, man-страница команды |
| `find` | **файлы и каталоги** по имени, типу, дате |

```bash
grep -r "TODO" project/       # текст в файлах
whereis grep                  # где лежит программа grep
find . -name "*.log"          # файлы по имени
```
