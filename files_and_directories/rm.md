# Команда `rm`

`rm` (remove) удаляет файлы и каталоги.

Утилита обычно находится в `/bin/rm`.

**Внимание:** удалённые файлы обычно **не попадают в корзину** и **не восстанавливаются** простой командой.

## Базовое использование

Удалить один файл:

```bash
rm file
```

Удалить несколько файлов:

```bash
rm file1 file2 file3
```

Удалить по шаблону:

```bash
rm *.txt
rm file?
```

Перед удалением полезно проверить список:

```bash
ls file1 file2
rm file1 file2
```

## Полезные опции

| Опция | Описание |
| ----- | -------- |
| `-r` | Рекурсивно: удалить каталог и всё внутри |
| `-f` | Не спрашивать и не показывать ошибки, если файла нет |
| `-i` | Спрашивать подтверждение перед каждым удалением |
| `-v` | Показать, что было удалено |
| `-d` | Удалить пустые каталоги (без `-r`) |

Опции можно комбинировать:

```bash
rm -rv dir
rm -rf dir
```

## Примеры

Удалить файл:

```bash
touch file
rm file
ls file
# ls: cannot access 'file': No such file or directory
```

Удалить с подтверждением:

```bash
rm -i file.txt
```

Удалить каталог с содержимым:

```bash
mkdir -p dir/subdir
touch dir/file dir/subdir/a
rm -r dir
```

Удалить только пустой каталог:

```bash
mkdir empty
rmdir empty
# или
rm -d empty
```

## `rm -r` и `rm -rf`

Каталог нельзя удалить обычным `rm` без `-r`:

```bash
mkdir dir
rm dir
# rm: cannot remove 'dir': Is a directory
```

Нужно рекурсивное удаление:

```bash
rm -r dir
```

`-f` (force) отключает вопросы и игнорирует отсутствующие файлы:

```bash
rm -rf dir
```

**Опасно:** `rm -rf` удаляет безвозвратно. Ошибка в пути может уничтожить важные данные:

```bash
rm -rf / tmp/foo   # пробел после / — катастрофа (не запускайте)
rm -rf ~/dir       # удалит ~/dir целиком
```

Перед `rm -rf` всегда проверяйте путь:

```bash
ls -la dir
rm -rf dir
```
