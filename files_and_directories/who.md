# Команда `who`

`who` показывает, **кто сейчас вошёл в систему** и откуда.

Утилита обычно находится в `/usr/bin/who`.

## Базовое использование

Показать активные сессии:

```bash
who
```

Типичный вывод на сервере:

```text
alex     tty1         2024-06-15 09:10
root     pts/0        2024-06-15 10:30 (192.168.1.5)
```

Колонки обычно означают:

| Колонка | Значение |
| ------- | -------- |
| имя пользователя | кто вошёл |
| терминал | `tty1`, `pts/0` и т.д. |
| дата и время | когда началась сессия |
| `(host)` | откуда подключились (если есть) |

## Полезные опции

| Опция | Описание |
| ----- | -------- |
| `-a` | Показать всю доступную информацию |
| `-b` | Время последней загрузки системы |
| `-q` | Только имена и число пользователей |
| `-H` | Добавить заголовки колонок |
| `-u` | Добавить время простоя (idle) |
| `-T` | Состояние терминала: `+` активен, `-` простаивает |

```bash
who -H
who -u
who -q
who -b
```

## Примеры

Кто в системе:

```bash
who
```

Краткий список:

```bash
who -q
# root
# # users=1
```

С заголовками и idle-временем:

```bash
who -Hu
```

Когда система была загружена:

```bash
who -b
# system boot  2024-06-15 08:00
```

## `who`, `whoami` и `w`

| Команда | Что показывает |
| ------- | -------------- |
| `who` | все активные входы в систему |
| `whoami` | только **ваше** имя пользователя |
| `w` | кто вошёл + чем занят (команды, нагрузка) |

```bash
whoami
# root

who
# root     pts/0   ...
```

## В Docker-контейнере

В контейнере вывод часто минимальный:

```bash
who
# root
```

Это нормально: в контейнере обычно нет полноценных пользовательских сессий, как на обычном сервере.
