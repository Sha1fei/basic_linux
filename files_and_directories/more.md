# Команда `more`

`more` постранично показывает **текстовый файл** — экран за экраном, только **вперёд**.

Утилита обычно находится в `/usr/bin/more`.

Проще и старше, чем [less](less.md): нельзя нормально листать назад (или возможности ограничены). Сегодня чаще используют `less`, но `more` всё ещё встречается.

## Базовое использование

Открыть файл:

```bash
more /var/log/syslog
```

Просмотр вывода команды через pipe:

```bash
ls -l /usr/bin | more
cat long_file.txt | more
```

Синтаксис:

```text
more [опции] [файл...]
```

## Управление внутри `more`

| Клавиша | Действие |
| ------- | -------- |
| `Пробел` | Следующая страница |
| `Enter` | Следующая строка |
| `q` | Выход |
| `h` | Справка |
| `/текст` | Поиск вперёд |
| `n` | Следующее совпадение поиска |

Листать **назад** в классическом `more` нельзя — для этого нужен `less`.

## Полезные опции

| Опция | Описание |
| ----- | -------- |
| `-N` | Показывать по N строк за раз (по умолчанию — размер экрана) |
| `-d` | Подсказки «Press space to continue» |
| `-s` | Сжимать несколько пустых строк в одну |
| `+N` | Начать с строки N |
| `+/шаблон` | Начать с первой строки, где есть шаблон |

```bash
more -10 file.txt       # по 10 строк
more +50 file.txt       # с 50-й строки
more +/error app.log    # с первой строки с error
```

## Примеры

Просмотр длинного файла:

```bash
more /usr/share/dict/words
```

Длинный вывод команды:

```bash
ps aux | more
find /etc -type f | more
```

Начать чтение с середины файла:

```bash
more +100 config.log
```

## `more` и `less`

| | `more` | `less` |
| --- | --- | --- |
| Листать назад | нет (или ограничено) | да |
| Поиск | базовый | полноценный |
| Использование | реже | **предпочтительнее** |
| Команда pipe | часто `... \| more` | `... \| less` |

```bash
man ls | more
man ls | less    # удобнее
```

На многих системах `more` — обёртка или алиас, который внутри вызывает `less`.
