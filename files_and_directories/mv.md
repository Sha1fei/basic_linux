# Команда `mv`

`mv` (move) перемещает или переименовывает файлы и каталоги.

Утилита обычно находится в `/bin/mv` или `/usr/bin/mv`.

В отличие от `cp`, `mv` **не оставляет копию** — объект переносится в новое место.

## Базовое использование

Переименовать файл:

```bash
mv old.txt new.txt
```

Переместить файл в каталог:

```bash
mv file.txt dir/
```

Переместить несколько файлов в каталог:

```bash
mv file1 file2 file3 dir/
```

Синтаксис:

```text
mv [опции] источник назначение
mv [опции] файл1 файл2 ... каталог
```

## Полезные опции

| Опция | Описание |
| ----- | -------- |
| `-i` | Спрашивать перед перезаписью |
| `-f` | Перезаписывать без вопросов |
| `-v` | Показать, что перемещается |
| `-u` | Перемещать только если источник новее или файл ещё не существует |
| `-n` | Не перезаписывать существующий файл |

Опции можно комбинировать:

```bash
mv -iv file.txt backup/
```

## Примеры

Переименовать файл:

```bash
touch file
mv file renamed
ls renamed
```

Переместить в каталог:

```bash
mkdir backup
touch file
mv file backup/
ls backup/
```

Переместить каталог:

```bash
mkdir dir
mv dir archive/
ls archive/
```

Для каталогов `-r` **не нужен** — в отличие от `cp`:

```bash
mkdir -p dir/subdir
touch dir/file
mv dir dir_moved
ls -R dir_moved
```

Спросить перед перезаписью:

```bash
mv -i file.txt backup/
```

## `mv` и `cp` — в чём разница

| | `mv` | `cp` |
| --- | --- | --- |
| Результат | файл переносится | создаётся копия |
| Переименование | да | только если указать новое имя |
| Каталоги | переносятся без `-r` | нужен `cp -r` |
| Исходный файл | исчезает | остаётся |

```bash
cp file copy    # file останется
mv file moved   # file исчезнет, появится moved
```
