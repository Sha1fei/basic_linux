# Команда `cd`

`cd` (change directory) переключает текущий рабочий каталог в оболочке.

В отличие от `ls`, `touch` и `mkdir`, `cd` — это **встроенная команда shell** (builtin), а не отдельная программа в `/bin`.

Проверить:

```bash
type cd
# cd is a shell builtin
```

## Базовое использование

Перейти в каталог:

```bash
cd dir
```

Перейти в домашнюю папку:

```bash
cd
cd ~
```

Перейти в корень файловой системы:

```bash
cd /
```

Узнать, где вы сейчас:

```bash
pwd
```

## Относительные и абсолютные пути

**Абсолютный путь** — от корня `/`:

```bash
cd /etc
cd /home/user/documents
```

**Относительный путь** — от текущей папки:

```bash
cd dir
cd dir/subdir
cd ../other
```

| Путь | Значение |
| ---- | -------- |
| `.` | текущий каталог |
| `..` | родительский каталог |
| `~` | домашняя папка |
| `-` | предыдущий каталог |

## Полезные примеры

На уровень выше:

```bash
cd ..
```

Вернуться в предыдущий каталог:

```bash
cd /var/log
cd /etc
cd -
# снова окажетесь в /var/log
```

В домашнюю папку другого пользователя (если есть доступ):

```bash
cd ~root
```

Перейти в каталог с пробелом в имени:

```bash
cd "my dir"
cd my\ dir
```

## Опции в Bash

| Опция | Описание |
| ----- | -------- |
| `-L` | Следовать символическим ссылкам (поведение по умолчанию) |
| `-P` | Не следовать ссылкам — перейти в физический каталог |

```bash
cd -P /path/to/symlink
```

## Примеры

Создать каталог и зайти в него:

```bash
mkdir dir
cd dir
pwd
```

Перейти в проект по абсолютному пути:

```bash
cd /work/basic_linux
pwd
```

Вернуться домой:

```bash
cd ~
pwd
```

## Где `cd` работает, а где нет

`cd` меняет каталог **только в текущей оболочке**. Дочерний процесс не может переключить родительский терминал.

```bash
cd /tmp
bash -c 'cd /etc && pwd'   # /etc — внутри подпроцесса
pwd                         # /tmp — ваш терминал не изменился
```

### В интерактивном терминале

Просто введите `cd` — этого достаточно:

```bash
cd /etc
pwd   # /etc
```

### В скрипте (для команд внутри скрипта)

`cd` внутри скрипта меняет каталог **для этого скрипта**:

```bash
#!/usr/bin/env bash
cd /var/log
ls
```

Запуск:

```bash
bash myscript.sh
```

После завершения скрипта ваш терминал останется там, где был.

### Чтобы скрипт сменил каталог в вашем терминале

Нужно **подключить** скрипт к текущей оболочке, а не запускать как отдельную программу:

```bash
source go_etc.sh
# или
. go_etc.sh
```

Содержимое `go_etc.sh`:

```bash
cd /etc
```

Тогда `cd` выполнится в **вашей** оболочке, и вы окажетесь в `/etc`.
