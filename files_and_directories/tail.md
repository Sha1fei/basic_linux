# Команда `tail`

`tail` выводит **последние строки** файла.

Утилита обычно находится в `/usr/bin/tail`.

Часто используют для просмотра свежих записей в логах и отслеживания изменений в реальном времени.

## Базовое использование

Последние 10 строк файла (по умолчанию):

```bash
tail /var/log/syslog
```

Несколько файлов:

```bash
tail app.log app.old.log
```

Синтаксис:

```text
tail [опции] [файл...]
```

Без файла `tail` читает **stdin**:

```bash
echo -e "one\ntwo\nthree" | tail -2
```

## Полезные опции

| Опция | Описание |
| ----- | -------- |
| `-n N` | Показать последние N строк |
| `-n +N` | Показать файл **начиная с** строки N до конца |
| `-c N` | Показать последние N **байт** |
| `-f` | Следить за файлом: показывать новые строки по мере появления |
| `-F` | Как `-f`, но ждёт появления файла, если его ещё нет |
| `-q` | Не печатать имена файлов |
| `-v` | Всегда печатать имена файлов |

Короткая запись:

```bash
tail -5 file.txt    # то же, что tail -n 5 file.txt
```

## Примеры

Последние 20 строк лога:

```bash
tail -n 20 /var/log/auth.log
tail -20 /var/log/auth.log
```

Файл с 100-й строки до конца:

```bash
tail -n +100 file.txt
```

Следить за логом в реальном времени:

```bash
tail -f /var/log/syslog
# Ctrl+C — выход
```

Следить за несколькими файлами:

```bash
tail -f app.log error.log
```

Дождаться появления файла и следить за ним:

```bash
tail -F /var/log/new_service.log
```

Последние 50 байт:

```bash
tail -c 50 file.bin
```

## `tail -f` в работе

Режим `-f` (follow) держит файл открытым и выводит новые строки — типичный сценарий при отладке:

```bash
tail -f /var/log/nginx/access.log
```

Полезные сочетания:

```bash
tail -f app.log | grep ERROR
tail -n 100 -f app.log    # сначала 100 строк, потом follow
```
