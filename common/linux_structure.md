# Базовая структура Linux

В Linux вся файловая система начинается с корневого каталога `/`.
Все остальные файлы, папки, устройства и подключенные диски находятся внутри него.

## Корневой каталог

`/` - главная точка файловой системы.

Пример просмотра содержимого корня:

```bash
ls /
```

Пример типичной структуры:

```text
/
├── bin
├── boot
├── dev
├── etc
├── home
├── lib
├── media
├── mnt
├── opt
├── proc
├── root
├── run
├── sbin
├── srv
├── sys
├── tmp
├── usr
└── var
```

## Назначение основных каталогов

### `/bin`

Основные пользовательские команды, которые нужны для работы системы и shell.

Примеры:

```text
ls
cp
mv
cat
echo
bash
```

### `/sbin`

Системные команды для администрирования.
Часто используются администратором или пользователем `root`.

Примеры:

```text
ip
reboot
shutdown
mount
```

### `/etc`

Конфигурационные файлы системы и установленных программ.

Примеры:

```text
/etc/passwd
/etc/hosts
/etc/ssh/
/etc/systemd/
```

### `/home`

Домашние каталоги обычных пользователей.

Например, у пользователя `alex` домашняя папка обычно будет:

```text
/home/alex
```

### `/root`

Домашний каталог суперпользователя `root`.
Это не то же самое, что `/`.

### `/usr`

Программы, библиотеки, документация и общие данные пользователя.
Большая часть установленного ПО обычно находится здесь.

Частые подкаталоги:

```text
/usr/bin
/usr/sbin
/usr/lib
/usr/share
```

### `/var`

Изменяемые данные системы: логи, кэши, очереди, базы некоторых сервисов.

Примеры:

```text
/var/log
/var/cache
/var/tmp
```

### `/tmp`

Временные файлы.
Содержимое этой папки может очищаться автоматически.

### `/dev`

Файлы устройств.
В Linux устройства представлены как файлы.

Примеры:

```text
/dev/null
/dev/random
/dev/sda
/dev/tty
```

### `/proc`

Виртуальная файловая система с информацией о процессах и ядре.
Файлы здесь не лежат на диске, а создаются системой во время работы.

Примеры:

```text
/proc/cpuinfo
/proc/meminfo
/proc/1
```

### `/sys`

Виртуальная файловая система с информацией об устройствах, драйверах и ядре.

### `/boot`

Файлы, необходимые для загрузки системы.

Примеры:

```text
kernel
initramfs
bootloader files
```

### `/lib` и `/lib64`

Системные библиотеки, которые нужны программам из `/bin` и `/sbin`.

### `/media`

Точка автоматического подключения съемных устройств.

Примеры:

```text
USB-флешки
внешние диски
карты памяти
```

### `/mnt`

Временная точка ручного подключения файловых систем.
Часто используется для тестового монтирования дисков или сетевых папок.

### `/opt`

Дополнительные сторонние программы.
Иногда сюда устанавливают крупные приложения, которые не входят в стандартную систему.

### `/srv`

Данные сервисов, которые предоставляет сервер.

Примеры:

```text
веб-сайты
FTP-данные
файлы серверных приложений
```

### `/run`

Временные данные работающей системы.
Здесь могут находиться PID-файлы, socket-файлы и служебная информация процессов.
